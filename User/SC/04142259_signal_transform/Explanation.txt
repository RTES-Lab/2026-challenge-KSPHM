[실험 목적]
  F2S2 논문의 Section 3 "Signal Transformation"만 분리하여 적용.
  RPM 변화에 의한 특징의 Signal Jump를 제거함으로써:
    1. 특징의 단조성(Monotonicity) 개선
    2. 특징의 추세성(Trendability) 개선
    3. 이후 HI→SSM 파이프라인에 깨끗한 입력 제공

[핵심 알고리즘: Signal Transformation]
  원본 신호 y_k^p (조건 p에서 관측) → 변환된 신호 y_k^B (baseline 조건으로):
    y_k^B = a'_p × y_k^p + b'_p

  a'_p, b'_p 추정:
    1. Baseline 조건 = 가장 빈번한 RPM 구간
    2. 조건 p 활성 시간에서 baseline 신호를 선형 보간
    3. Least Squares: min Σ(y_interp - a'_p * y_p - b'_p)²
    4. baseline 조건은 항등 변환 (a'=1, b'=0)

[확인된 문제점]
  1. 전체 개선폭이 제한적 (+5.8%)
     - 3구간 이산화가 너무 거칠 수 있음 (실제 RPM은 연속적)
     - 현재 분위수 경계 [669, 740, 967, 990]: 중간 구간이 너무 넓음(740~967)
     - RPM 변동의 실제 패턴(1시간 단위 계단)에 맞는 이산화 필요

  2. 일부 특징 악화
     - kurt_rms, skewness 등 통계적 고차 모멘트 특징은
       비선형 신호 특성상 선형 변환(a'y+b')이 부적합
     - 이런 특징은 별도 처리(로그 변환 후 선형 보정 등) 필요

  3. Baseline 조건 선택 논의
     - 고속 RPM이 baseline으로 선택됨
     - 논문 원문: "가장 오래 작동한 조건"을 baseline으로 권장
     - RPM이 균등 분포이면 어느 걸 선택해도 유사

  4. 보간의 한계
     - baseline 구간이 sparse한 시간대에서 보간 정확도 저하
     - 특히 수명 초기/말기에 baseline 데이터가 적으면 변환 파라미터 왜곡
